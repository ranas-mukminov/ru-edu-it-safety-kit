# Безопасность и Приватность

## Модель угроз
Этот проект ориентирован на защиту от базовых угроз, актуальных для школ:
1.  **Доступ к нежелательному контенту**: Блокировка порнографии, экстремизма, наркотиков (через DNS).
2.  **Вредоносное ПО**: Изоляция зараженных устройств через VLAN.
3.  **Неавторизованный доступ**: Разделение сетей (Ученики не видят принтеры Учителей).

## Ограничения
- **Не является средством обхода блокировок**: Мы не предоставляем инструментов для обхода РКН/ТСПУ.
- **Отсутствие секретов**: В репозитории нет реальных паролей. Все примеры содержат заглушки.
- **Юридическая сила**: Все документы — шаблоны. Требуется утверждение директором школы.

## Безопасная разработка
- Используется `yaml.safe_load` для защиты от RCE при парсинге профилей.
- Зависимости проверяются через `pip-audit`.
