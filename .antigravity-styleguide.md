# Antigravity Styleguide for ru-edu-it-safety-kit

## Code Style
- **Python**: Follow PEP 8.
  - Use `snake_case` for functions and variables.
  - Use `CamelCase` for classes.
  - Use type hints for all public functions and classes.
  - Docstrings: Google style or NumPy style.
- **Formatting**: Use `black` and `ruff`.
  - Line length: 120 characters.

## Testing
- **Framework**: `pytest`.
- **Coverage**: Aim for high coverage on core logic (generators, validators).
- **Discipline**:
  - Write tests *before* or *alongside* code (TDD).
  - Ensure tests are deterministic and fast.
  - Mock external dependencies (filesystem, network).

## Security
- **Secrets**: NEVER commit secrets. Use environment variables or placeholders.
- **Input Validation**: Validate all YAML inputs using Pydantic models.
- **Safe Parsing**: Use `yaml.safe_load`.
- **No Evasion**: Do not implement features designed to bypass legal blocking (DPI/RKN).

## AI Agent Guidance
- When editing this repo:
  - Keep tests green.
  - Add tests for new features.
  - Prefer explicit code over "clever" one-liners.
  - Respect the project's goal: IT safety for schools, not hacking tools.
